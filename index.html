DOCX conversion

(OOXML requires a PNG fallback next to every SVG, so a vector DOCX contains both and is larger. Kindle reflowable books normally show the PNG — vector pays off in Word and in print. "Keep original" avoids re-encoding, so JPEG photos stay JPEG instead of ballooning into PNG.)

EPUB (recommended for Kindle)

(KDP prefers EPUB 3 and converts it to KFX with the least loss. Unlike DOCX, images keep their own pixels — a DOCX caps figure quality at its on-page inches, which is what makes figures look blurry. Kindle's Enhanced Typesetting does not render SVG, so figures are embedded as high-resolution raster, sized by CSS.)

Join chapter HTML back into one file

No folder selected

(Pick the book/ folder produced by htmlbook.py split. The chapters are reassembled into the single HTML using manifest.json for order. Nothing is written until you press Save as…, which lets you name the file or overwrite the original. The result is also handed to the conversion tab, so you can read it through and convert it without reloading anything. htmlbook.py stays the authority for the archival round-trip; this button is the review loop.)

Split a single HTML into chapter files

No file selected

(Chapters are cut at every h1/h2 — the same rule as htmlbook.py split. Before anything is written, the result is re-joined in the browser and compared byte-for-byte with the original; if it does not round-trip, nothing is written. Press Write book folder… to save _frame.html, manifest.json and the chNN_slug.html files. If your browser cannot write to a folder, a book.zip is downloaded instead. htmlbook.py stays the authority for archival splits; this tab is the review loop.)
